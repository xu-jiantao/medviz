设计一套医疗统计图形需求的UI界面和菜单架构。适合作为临床决策支持系统（CDSS）或医疗数据可视化看板的核心模块。
为了兼顾界面的专业性与易用性，建议采用“左侧导航菜单 + 顶部工具栏 + 中间主视觉区 + 右侧临床辅助决策面板”的经典响应式布局。
以下是具体的UI界面设计与菜单按钮规划：
一、 整体页面布局 (UI Layout)
* 左侧导航栏 (Sidebar Menu)： 按照图形类别划分一级菜单，按照应用场景划分二级菜单，方便登录用户快速定位。
* 顶部控制栏 (Top Action Bar)： 放置全局信息，如当前患者信息（姓名、床号、诊断）、全局时间范围筛选器、以及导出/打印报告(Exce或PDF)按钮。
* 核心图表区 (Main Chart Area)： 占据屏幕中央空间，用于展示具体的趋势图、雷达图、热图或列线图。
* 右侧分析面板 (Right Clinical Panel)： 用于动态显示当前图表的“临床判断”结论及参数微调表单。
二、 菜单与按钮详细设计
结合文档中的四大类图形需求，左侧导航栏及对应界面的功能按钮可设计如下：
1. 趋势图模块 (Trend Charts)

核心用途： 追踪同一指标随时间的变化，识别异常波动、疗效反应或病情进展 。
* 左侧菜单结构：
    *  趋势分析
        * 单指标纵向趋势 
        * 多指标叠加趋势 
        * 治疗/方案对比趋势 
* 顶部/图表内工具栏按钮：
    * 
[设置 X/Y 轴]：允许用户自定义横轴（时间刻度）和纵轴（指标单位） 。 
    * 
[添加标记节点]：用于快速插入用药调整（加药、换药、停药）或特殊事件节点 。 
    * 
[设置参考线/区间]：用于添加诊断阈值或目标范围的阴影区域 。 
* 
右侧面板展示： 自动输出如“WBC下降但PCT持续升高，提示可能存在非细菌性感染”等临床判断预警 。 
2. 雷达图模块 (Radar Charts)

核心用途： 多维度综合评估，识别“短板”维度 。
* 左侧菜单结构：
    * 综合评估
        * 生理评分 (APACHE II / SOFA) 
        * 专科评估 (心功能 / 肿瘤 / 术前) 
        * 动态对比 (康复进程) 
* 顶部/图表内工具栏按钮：
    * 
[自定义维度]：允许医生增加或减少评估的轴数（维度数量及规则自定义） 。 
    * 
[添加对比基线]：例如在康复评估中，叠加显示“入院时”与“出院时”的两张图层 。 
* 
右侧面板展示： 突出显示外凸严重的轴（如“氧合”和“GCS”），并给出应对建议 。 
3. 热图模块 (Heatmaps)

核心用途： 矩阵式数据可视化，通过颜色深浅快速识别异常值或聚类 。
* 左侧菜单结构：
    * 矩阵热图
        * 检验/生命体征矩阵 
        * 时间-事件追踪 
        * 影像/病理组学 
        * 基因/药敏精准医疗 
* 顶部/图表内工具栏按钮：
    * 
[行列维度配置]：下拉菜单选择行与列代表的数据集（如行选患者，列选时间或特征） 。 
    * 
[颜色映射设置]：调整颜色阈值，例如设置IC50值（绿色=敏感，红色=耐药） 。 
    * 
[添加事件参考线]：如在体温热图中叠加抗生素给药时间的黑色竖线 。 
4. 列线图模块 (Nomograms)

核心用途： 复杂统计模型可视化计算尺，实现床旁快速、个体化的风险预测 。
* 左侧菜单结构：
    *  风险预测模型
        * 生存/预后预测 
        * 并发症/不良事件预测 
        * 诊断概率评估 
* 主视觉区交互设计：
    * 
表单输入模式： 左侧提供年龄、分期、淋巴结等指标的下拉框或滑动条输入 。 
    * 
动态列线图展示： 用户在表单输入后，右侧的列线图上自动生成垂直线（画线到"Points"轴）并计算总分及对应概率 。 
* 顶部/图表内工具栏按钮：
    * 
[切换底层模型]：允许用户在不同的统计模型（如Logistic回归、Cox回归）之间无缝切换 。
